## Exercise 1 — SELECT all columns

```sql
SELECT * FROM books;
```

**Explanation:** `SELECT *` means "all columns." The `*` is a shorthand for listing every column in the order they were defined. In production code, list columns explicitly (`SELECT id, title, author`) so the query is not affected by schema changes (added or reordered columns).

---

## Exercise 2 — SELECT with WHERE

```sql
SELECT id, title, author, year
FROM   books
WHERE  available = true
ORDER  BY year ASC;
```

**Explanation:**
- `WHERE available = true` — only rows where `available` is exactly `true` (not NULL, not false)
- `ORDER BY year ASC` — oldest first (ASC is the default, so `ORDER BY year` is equivalent)
- Three rows match: 1984 (1949), Dune (1965), Neuromancer (1984)

**Alternative for boolean columns in PostgreSQL:**
```sql
WHERE available          -- equivalent to WHERE available = true
WHERE NOT available      -- equivalent to WHERE available = false
```

## Exercise 3 — Filter with AND/OR

```sql
SELECT id, title, author, year, available
FROM   books
WHERE  (available = true AND year > 1960)
    OR  author = 'Isaac Asimov'
ORDER  BY year;
```

**Explanation:**
- Parentheses clarify precedence: `AND` binds tighter than `OR` in SQL, but explicit parentheses are always clearer
- Result: Foundation (Asimov, regardless of availability), Dune (available + > 1960), Neuromancer (available + > 1960)

**Common mistake:** Forgetting parentheses when mixing AND/OR. `A AND B OR C` is evaluated as `(A AND B) OR C`, which may not be what you intend.

## Exercise 4 — INSERT with RETURNING

```sql
INSERT INTO books (title, author, year, available)
VALUES ('The Left Hand of Darkness', 'Ursula K. Le Guin', 1969, true)
RETURNING id, title;
```

**Explanation:**
- Column list `(title, author, year, available)` explicitly names the columns being populated
- `id` is omitted — it is auto-generated by `BIGSERIAL`
- `RETURNING id, title` fetches the generated `id` in the same round-trip — no second query needed

**Multi-row INSERT:**
```sql
INSERT INTO books (title, author, year) VALUES
    ('Solaris',    'Stanislaw Lem',     1961),
    ('Hyperion',   'Dan Simmons',       1989),
    ('Blindsight', 'Peter Watts',       2006)
RETURNING id, title;
```

## Exercise 5 — UPDATE single row

```sql
UPDATE books
SET    available = true
WHERE  id = 3
RETURNING id, title, available;
```

**Explanation:**
- `WHERE id = 3` targets exactly one row because `id` is the primary key (unique)
- `SET available = true` changes only the `available` column; all other columns are unchanged
- `RETURNING` confirms the new state in one operation

**Without RETURNING:**
```sql
UPDATE books SET available = true WHERE id = 3;
-- Then separately verify:
SELECT id, title, available FROM books WHERE id = 3;
```

## Exercise 6 — UPDATE multiple rows

```sql
UPDATE books
SET    available = false
WHERE  year < 1960
RETURNING id, title, year, available;
```

**Explanation:**
- Three books match the condition: Foundation (1951), 1984 (1949), Brave New World (1932)
- All three are updated in a single statement — no loop required
- `RETURNING` shows all affected rows

**Safe multi-row UPDATE pattern:** Always preview with a SELECT using the same WHERE clause before the UPDATE.

## Exercise 7 — DELETE with RETURNING

```sql
DELETE FROM books
WHERE  id = 5
RETURNING id, title;
```

**Explanation:**
- Deletes exactly one row (PK lookup)
- `RETURNING` confirms what was deleted (useful for audit logging)
- After deletion, id=5 is gone permanently unless you re-insert

**Note:** Gaps in `id` values are normal and expected. IDs are not recycled.

## Exercise 8 — Aggregates

```sql
SELECT
    COUNT(*)                           AS total_books,
    COUNT(*) FILTER (WHERE available)  AS available_books,
    MIN(year)                          AS oldest_year,
    MAX(year)                          AS newest_year,
    ROUND(AVG(year))                   AS avg_year
FROM books;
```

**Explanation:**
- `COUNT(*)` — counts all rows including those where `available` is false
- `COUNT(*) FILTER (WHERE available)` — counts only rows where `available = true` (PostgreSQL extension, cleaner than COUNT + CASE)
- `MIN`, `MAX`, `AVG` — standard aggregate functions
- `ROUND(AVG(year))` — AVG returns NUMERIC by default; ROUND removes fractional years

**Alternative for available count:**
```sql
SUM(CASE WHEN available THEN 1 ELSE 0 END) AS available_books
```
The FILTER approach is cleaner and preferred in PostgreSQL.
